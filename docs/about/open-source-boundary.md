# Open Source Boundary

Dayan AI Workshop publishes reusable governance patterns, not private operating records.

## Public

Safe to publish:

- general methods;
- glossary definitions;
- template checklists;
- permission patterns;
- verifier examples;
- redacted scenarios;
- tool comparison frameworks.

## Client-Redacted

Can publish only after redaction:

- project walkthroughs;
- before / after workflow diagrams;
- sample reports;
- screenshots;
- workflow metrics.

Required redaction:

- remove names, contact details, contracts, internal project paths, screenshots with identifying data, and any specific commercial terms.

## Internal

Do not publish:

- private operating notes;
- personal memory;
- team-specific tactics;
- unreleased strategy;
- detailed automation routes that expose security posture.

## Secret

Never publish:

- API keys;
- OAuth tokens;
- passwords;
- private keys;
- seed phrases;
- environment files;
- signed transaction material;
- customer secrets.

## Principle

Open the architecture. Protect the people, customers, and systems behind it.
